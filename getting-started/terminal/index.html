The Terminal

If at some point you've seen the screen of a programmer at work, it probably looks something like this:

To non-programmers, this screen looks pretty much the same as this:

But don't worry! One of the first things we'll do is get you acccustomed to working in the command-line environment. The program in the first picture is called a Terminal. The name comes from the days of computer when you used a physical monitor and keyboard connected to a mainframe, that looked something like this:

As computers advanced in capability, you could run a terminal emulator, which just a program that emulates the physical terminal. Over the years, the use of a command-line interface has still proved to be useful for programmers.

Mac OS X comes with a terminal program named Terminal and it works fine, but I recommend a free alternative called iTerm 2. Download iTerm 2 from www.iterm2.com and copy the application into your Applications folder. Once you launch iTerm, it will look something like this:

Notice the cursor is positioned just after the $ character. This is called the prompt. You can type something there and press enter and the terminal will run your program, display the output. Throughout the rest of this book, when you should do something at the command-line, you will see this:
